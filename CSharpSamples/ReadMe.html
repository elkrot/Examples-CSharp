Примеры Visual Studio 2010 для C# 4.0

Примеры C# подразделяются на две категории:

  • Примеры использования языка программирования
  • Примеры LINQ

Примеры использования языка программирования

Примеры использования языка программирования приводятся для ознакомления с различными возможностями языка C#. Они находятся в папке LanguageSamples.

 Примеры, характерные для C# 4.0, называются следующим образом:

  • Именованные и необязательные
  • Пример для Office
  • Пример для Python
  • Простая дисперсия.

Вот полный список примеров использования языка программирования:

  • Безымянные делегаты: Демонстрируется использование делегатов без имени для снижения сложности приложения.
  • Массивы: Показано использование массивов.
  • Атрибуты: Показано, как создавать классы особых атрибутов, использовать их в коде и посылать к ним запросы через отражение.
  • Классы-коллекции: Показано, как создавать классы-коллекции, не являющиеся универсальными, которые могут использоваться с оператором forEach.
  • COM-взаимодействие, часть I: Показано взаимодействие с объектами COM в C#.
  • COM-взаимодействие, часть II: Показано, как использовать сервер C# с клиентом COM C++.
  • Командная строка: Демонстрируется обработка простой командной строки и индексация массива.
  • Условные методы: Демонстрируются условные методы - мощный механизм, посредством которого обращения к методам могут добавляться или пропускаться в зависимости от того, определен ли символ.
  • Делегаты: Показано, как объявлять делегаты, сопоставлять их со статическими методами и методами экземпляров и объединять в множественные делегаты.
  • События: Показано, как в C# объявлять и вызывать события, а также задавать их конфигурацию.
  • Явный интерфейс: Демонстрируется, как явно реализовывать члены интерфейса и как обращаться к этим членам из экземпляров интерфейса.
  • Универсальные шаблоны: Показано, как создавать универсальные классы-коллекции, которые могут использоваться с оператором forEach.
  • Здравствуй, мир!: Приложение "Здравствуй, мир!"
  • Индексаторы, часть I: Показано, как классы C# могут объявлять индексаторы для получения доступа к объектам как к массивам.
  • Индексаторы, часть II: Показано, как реализовать класс, использующий проиндексированные свойства. Проиндексированные свойства позволяют использовать класс, представляющий коллекцию типа массив.
  • Библиотеки: Показано использование параметров компилятора для создания DLL из нескольких исходных файлов, а также использование библиотеки в других программах.
  • Именованные и необязательные (C# 4.0): Демонстрируются именованные и необязательные параметры как альтернатива перегрузке методов.
  • Обнуляемые типы (типы, допускающие значение Null): Демонстрируются типы значений, например, тип Double и Boolean, которым может быть присвоено значение "Null".
  • Пример для Office (C# 4.0): Демонстрируется, как Dynamic и COM-взаимодействие упрощают вызов Microsoft Office в C# 4.0.
  • OLEDB (OLE для баз данных): На примере создания набора данных и добавления к нему таблиц демонстрируется использование в С# баз данных Microsoft Access.
  • Перегрузка операторов: Показано, как определенные пользователем классы могут перегружать операторы.
  • Разделяемые типы: Демонстрируется, как можно определять классы и структуры в нескольких файлах исходного кода C#.
  • PInvokeПоказано, как в C# вызывать экспортируемые из DLL функции.
  • Свойства: Показано, как объявлять и использовать функции. Кроме того, демонстрируются абстрактные свойства.
  • Пример для Python (C# 4.0): Дано объяснение, как вызывать скрипт Python, пользуясь функцией Dynamic в C# 4.0.
  • Безопасность: Обсуждается безопасность в .NET Framework; показаны два способа изменить разрешения системы безопасности в C#: с использованием классов разрешения и с использованием атрибутов разрешения доступа.
  • Простая дисперсия (C# 4.0): Показано, как Covariance и Contravariance поддерживаются интерфейсами и делегатами универсального типа.
  • Структуры: Показано, как использовать структуры в C#.
  • Потоки: Демонстрируются различные действия с потоками, например, создание и выполнение потока, синхронизация потоков, взаимодействие между потоками, использование пула потоков.
  • Небезопасный код: Показано, как использовать в C# неуправляемый код (код с указателями).
  • Пользовательские преобразования: Показано, как задавать преобразования к и из определённых пользователем типов.
  • Управление версиями: Демонстрируется управление версиями в C# посредством переопределения и новых ключевых слов.
  • XML-документы: Показано, как документировать код с использованием XML.
  • Yield: Демонстрируется, как использовать ключевое слово Yield для фильтрации элементов коллекции.

Примеры LINQ

Эти примеры помогают быстро освоить LINQ. Они также могут использоваться как справочные материалы для тех, кто уже имеет представление о LINQ. Сюда же включены и несколько полезных программ. Они находятся в папке LanguageSamples.

  • DynamicQuery: Код для создания запросов LINQ во время выполнения.
  • LinqToNorthwind: Основной пример использования LINQ To SQL для запроса к базе данных.
  • LinqToXmlDataBinding: Привязка кода LINQ to XML к средствам управления WPF.
  • ObjectDumper: Утилита для вывода на экран в текстовом режиме результатов запроса LINQ.
  • PasteXmlAsLinq: Дополнительный компонент Visual Studio, автоматически преобразующий XML к XML to LINQ.
  • QueryVisualizer: Позволяет разработчикам LINQ to SQL увидеть SQL своего запроса, а также результаты запроса в сетке.
  • Рефлектор: Использование LINQ для создания корректных запросов к объектам вашего кода при помощи Reflection API.
  • RSS: Этот образец действует как крошечный веб-сервер, агрегирующий несколько RSS-каналов.
  • SampleQueries: Это самый важный образец, который содержит около 500 примеров использования операторов запроса в LINQ to Objects, LINQ to SQL, LINQ to XML и LINQ to DataSet.
  • SimpleLambdas: Несколько примеров написания и использования лямбда-выражений.
  • SimpleLinqToObjects: Примеры LINQ начального уровня. Здесь показано, как легко делать простые запросы к объектам в памяти посредством LINQ to Objects.
  • SimpleLinqToXml: Начало использования LINQ to XML.
  • WebServiceLinqProvider(TerraService): Специальный поставщик LINQ и клиент веб-службы TerraServer-USA.
  • Whitepapers: В этой директории хранятся следующие документы в формате Word:
    • Обзор проекта LINQ
    • Обзор LINQ to SQL для C# и VB
    • Стандартные операторы запросов
    • Обзор LINQ to XML
  • WinFormsDataBinding: Инструкции по выводу запросов LINQ с сетками в форме Windows. Сюда включен пример запроса "один ко многим".
  • XQuery: Еще один простой пример запроса LINQ to XML. В этом примере показан код, необходимый для написания запроса LINQ to XML.

Проблемы при подключении к базе данных примеров LINQ

Строка подключения для примеров, используемая по умолчанию, должна работать без изменения, если на компьютере для разработки установлен экспресс-выпуск SQL Server. Ниже приведен вариант строки подключения, которую при необходимости можно изменить, если экспресс-выпуск SQL Server не используется, но доступен SQL Server. Возможно, в строках подключения примеров потребуется изменить слово "test" везде, где оно встречается.

string connString = 
  "server=test;database=northwind;user id=test;password=test";

В некоторых вариантах установки экспресс-выпуска SQL Server у вас, возможно, не будет прав на запуск пользовательского экземпляра. При получении соответствующего сообщения об ошибке попробуйте удалить из строки подключения конструкцию ";user instance = true".

Убедитесь, что копии базы данных "Борей", к которым осуществляется доступ, не помечены, как доступные только для чтения. При необходимости найдите копию файла NORTHWIND.MDF, к которой нужно получить доступ, щелкните этот файл правой кнопкой, выберите в контекстном меню пункт "Свойства". Снимите флажок атрибута Только чтение.

На некоторых маломощных компьютерах или при использовании виртуальных ПК пользователи сообщают о том, что при первом запуске некоторые приложения базы данных выдают ошибки. Попробуйте запустить пример два или три раза.

Настроенные копии базы данных "Борей", которыми комплектуются примеры, поставляются без файла журнала (LDF). Это сделано специально, чтобы не нарушить синхронизацию файлов LDF и MDF. При запуске примера, использующего базу данных, или при подключении к базе данных из Server Explorer в среде Visual Studio файл LDF будет создан автоматически. При подключении к файлу MDF возможны проблемы, если программа SQL Server Management Studio используется без предварительно созданного файла журнала.
