В этом примере запускается рабочий процесс, как только документ отправляется в 
библиотеку "Общие документы". Данный рабочий процесс создает запись в списке 
задач. Рабочий процесс завершается, когда пользователь указывает 100%-ное 
завершение задачи.

Для запуска примера необходимо выполнить следующие действия. 

- Изменить свойство "URL-адрес сайта" проекта на действительный сайт SharePoint 
  в вашей системе разработки (например: http://<имя_компьютера>)

Если требуется изменить свойство целевого списка или другие свойства 
конфигурации рабочего процесса, можно открыть мастер рабочих процессов 
SharePoint, щелкнув кнопку в свойстве "Конечный список" рабочего процесса.
